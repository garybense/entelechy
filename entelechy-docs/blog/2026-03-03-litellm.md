## The Problem: Stateless LLM Calls

You build an app with an LLM. User talks to it on Monday. Comes back Tuesday. The LLM has no idea who they are.

This is true for every provider. OpenAI, Anthropic, Groq, Azure — every API call is a blank slate. The LLM doesn't remember preferences, past conversations, or anything you've discussed before.

Most teams work around this by stuffing the last N messages into the context window. But that's not memory — it's a sliding window that drops everything older than your token limit. The user's preferences from last week? Gone. The project context from last month? Gone.

What if every LLM call automatically had the right context from past conversations?

---

## The Fix: Three Lines, Any Provider

`entelechy-litellm` hooks into LiteLLM to intercept every LLM call. Before the call, it retrieves relevant memories from Entelechy and injects them into the prompt. After the call, it stores the conversation for future retrieval.

```
Your code ──→ entelechy_litellm.completion()
                    │
                    ├─→ 1. Recall/Reflect from Entelechy (relevant memories)
                    ├─→ 2. Inject memories into prompt
                    ├─→ 3. Forward to LLM (any provider via LiteLLM)
                    ├─→ 4. Store conversation to Entelechy
                    └─→ 5. Return response
```

Here's the setup:

```python
import entelechy_litellm

entelechy_litellm.configure(
    entelechy_api_url="http://localhost:8888",
)

entelechy_litellm.set_defaults(
    bank_id="my-agent",
)

entelechy_litellm.enable()
```

That's it. Now every `completion()` call has memory:

```python
response = entelechy_litellm.completion(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Help me with my Python project"}],
    entelechy_query="What do I know about the user's Python project?",
)
```

The `entelechy_query` tells Entelechy what to search for in memory. The response comes back with context from past conversations — the LLM knows about the user's project, their preferences, their tech stack.

> **Note:** You can also use [Entelechy Cloud](https://ui.mindmods.org/signup) and skip the self-hosted setup entirely.

## Any Provider, Same Memory

Because the integration runs through LiteLLM, it works with every provider LiteLLM supports. Switch models freely — memory follows:

```python
messages = [{"role": "user", "content": "What did we discuss last time?"}]
query = "What have I discussed with this user?"

# OpenAI
entelechy_litellm.completion(model="gpt-4o", messages=messages, entelechy_query=query)

# Anthropic
entelechy_litellm.completion(model="claude-sonnet-4-20250514", messages=messages, entelechy_query=query)

# Groq
entelechy_litellm.completion(model="groq/llama-3.1-70b-versatile", messages=messages, entelechy_query=query)

# Azure OpenAI
entelechy_litellm.completion(model="azure/gpt-4", messages=messages, entelechy_query=query)

# AWS Bedrock
entelechy_litellm.completion(model="bedrock/anthropic.claude-3", messages=messages, entelechy_query=query)

# Google Vertex AI
entelechy_litellm.completion(model="vertex_ai/gemini-pro", messages=messages, entelechy_query=query)
```

Same bank, same memories, any model. Migrate from OpenAI to Anthropic and your agent still knows everything.

## Two Memory Modes

### Recall: Raw Memories

The default mode retrieves individual memory facts and injects them as a numbered list:

```python
entelechy_litellm.set_defaults(bank_id="my-agent", use_reflect=False)
```

The LLM sees something like:

```
# Relevant Memories
1. [WORLD] User is building a FastAPI application
2. [WORLD] User prefers pytest for testing
3. [OBSERVATION] User likes type hints
```

Best for precise, factual context where the LLM needs individual data points.

### Reflect: Synthesized Context

Reflect mode synthesizes memories into a coherent paragraph instead of raw facts:

```python
entelechy_litellm.set_defaults(
    bank_id="my-agent",
    use_reflect=True,
    reflect_context="I am a coding assistant helping with Python projects.",
)
```

The LLM sees something like:

```
The user is an experienced Python developer working on a FastAPI application.
They prefer pytest for testing and value type hints. In past conversations,
they asked about async patterns and database migrations.
```

Best for natural, conversational context. The `reflect_context` parameter shapes how Entelechy reasons about the memories without affecting what it retrieves.

## Direct Memory APIs

Sometimes you want to query memory outside of an LLM call. The direct APIs give you full control:

### Recall — query raw memories

```python
from entelechy_litellm import recall

memories = recall("what projects is the user working on?", budget="mid")
for m in memories:
    print(f"- [{m.fact_type}] {m.text}")
```

### Reflect — synthesized context

```python
from entelechy_litellm import reflect

result = reflect(
    "what do you know about the user's preferences?",
    context="I am a customer support agent.",
)
print(result.text)
```

### Retain — store memories manually

```python
from entelechy_litellm import retain

retain(
    content="User mentioned they're switching from Flask to FastAPI",
    context="Discussion about web frameworks",
)
```

All three have async variants: `arecall()`, `areflect()`, `aretain()`.

## Per-Call Overrides

Defaults are convenient, but sometimes you need per-call control:

```python
response = entelechy_litellm.completion(
    model="gpt-4o-mini",
    messages=[...],
    entelechy_query="What do I know about Alice?",
    entelechy_bank_id="other-agent",          # Different bank for this call
    entelechy_budget="high",                  # More thorough retrieval
    entelechy_reflect_context="Currently helping with onboarding",
)
```

Any default can be overridden with a `entelechy_*` prefix.

## Native SDK Wrappers

If you use the OpenAI or Anthropic SDKs directly (without LiteLLM), there are native wrappers:

### OpenAI

```python
from openai import OpenAI
from entelechy_litellm import wrap_openai

client = OpenAI()
wrapped = wrap_openai(
    client,
    bank_id="my-agent",
    entelechy_api_url="http://localhost:8888",
)

response = wrapped.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "What do you know about me?"}]
)
```

### Anthropic

```python
from anthropic import Anthropic
from entelechy_litellm import wrap_anthropic

client = Anthropic()
wrapped = wrap_anthropic(
    client,
    bank_id="my-agent",
    entelechy_api_url="http://localhost:8888",
)

response = wrapped.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello!"}]
)
```

Same memory, no LiteLLM dependency.

## Context Manager

For temporary memory integration:

```python
from entelechy_litellm import entelechy_memory
import litellm

with entelechy_memory(
    entelechy_api_url="http://localhost:8888",
    bank_id="user-123",
):
    response = litellm.completion(
        model="gpt-4o",
        messages=[{"role": "user", "content": "Hello!"}],
        entelechy_query="greeting context",
    )
# Memory integration automatically disabled after the block
```

## Bank Missions

Tell Entelechy what kind of knowledge the bank should build. This shapes how memories are consolidated into mental models. Pass `mission` and `bank_name` to `configure()`:

```python
entelechy_litellm.configure(
    entelechy_api_url="http://localhost:8888",
    mission="""This agent routes customer support requests.
    Remember which issue types go to which teams (billing, technical, sales).
    Track customer preferences and past resolutions.""",
    bank_name="Customer Support Router",
)
```

- Works with 100+ providers via LiteLLM, plus native OpenAI and Anthropic wrappers
